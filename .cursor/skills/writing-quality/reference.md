# Riferimento qualità scrittura

Regole dettagliate per l&#39;abilità `writing-quality`. Il file SKILL.md contiene le regole più comuni; questo file fornisce il riferimento completo per la punteggiatura, la terminologia, gli esempi di utilizzo delle maiuscole e la formattazione delle procedure.

---

## Punteggiatura

### Periodi

- Utilizzare i punti in un elenco se gli elementi completano una frase (&quot;Dal menu X, è possibile:&quot;)
- Utilizzare i periodi in un elenco se un elemento è una frase completa
- I periodi non devono essere formattati con il contenuto principale (Correzione: fare clic su **Fine**. / Non corretto: fare clic su **Fine.**)
- Utilizzare i punti nelle estensioni file (.pdf)

### Virgole

- Oxford virgola sempre: &quot;movente, mezzi e opportunità&quot;
- Virgola dopo la frase introduttiva: &quot;Per aprire un documento, fare clic su Apri.&quot;
- Virgola dopo l&#39;anno in una data a metà frase: &quot;Il 2 gennaio 2016, numero di...&quot;
- Virgola prima di &quot;then&quot; nelle procedure: &quot;Fare clic su **Configurazione**, quindi su **Interfaccia**.&quot;
- Nessuna virgola solo con mese-anno: &quot;Gennaio 2016&quot;

### Apostrofi

- Uso nelle contrazioni: no, sono, è
- Singolare possessivo: aggiungi (computer di Jen)
- Fine plurale possessiva in s: aggiungi solo &#39; (computer Joneses&#39;)
- Distinguere &quot;it&#39;s&quot; (è) da &quot;its&quot; (appartenente ad esso)

### Due punti

- Minuscolo dopo due punti a meno che non sia seguito da una citazione diretta, un nome corretto o un elenco puntato
- Utilizza alla fine di una frase/frammento che introduce un elenco
- Nessun due punti dopo i titoli della procedura
- Nessun due punti per introdurre elementi visivi o segmenti di codice

### Punti e virgola

- Connettere due clausole senza una congiunzione
- Frasi separate contenenti la punteggiatura corrispondente
- Non applicare il grassetto al punto e virgola dopo il testo in grassetto nelle routine
- Non includere in un collegamento ipertestuale

### Virgolette

- Fai riferimento a esse come &quot;virgolette&quot; (non &quot;virgolette&quot;)
- Le virgolette di chiusura vanno oltre le virgole e i periodi
- Il posizionamento del punto interrogativo dipende dal fatto che la domanda sia la citazione o la frase

### Trattini

- Se possibile, evita le strutture che richiedono trattini
- Sillabare i modificatori composti prima di un nome: &quot;memoria di sola lettura&quot;
- Sillabazione quando un modificatore è un participio: &quot;copia-protetto disco&quot;
- Sillabazione quando un modificatore è un numero: &quot;font a 12 punti&quot;
- Nessun trattino nelle combinazioni di tasti — usa + (Ctrl+I)

### Trattini lungo

Utilizza con moderazione. Considera invece la ristrutturazione della frase.

### Trattini lunghi

- Intervalli: 2005-2013
- Segno meno o numeri negativi
- Connettere l&#39;aggettivo composto a un&#39;altra parola: &quot;finestra di dialogo-opzioni&quot;

### Ellissi

- Codice o sintassi omessi
- Indicazione dell’interfaccia utente per la visualizzazione di una finestra di dialogo
- Testo omesso tra virgolette (aggiungere un punto se si sostituisce la fine della frase)

### Barra

- Nessuna barra per le scelte: usa &quot;o&quot; (&quot;lui o lei&quot; non &quot;lui/lei&quot;)
- Usare per combinazioni: interruttore on/off
- Uso per indirizzi Internet e frazioni

## Esempi di utilizzo delle maiuscole

### Termini Workfront: riferimenti diretti e indiretti

| Termine | Diretto (maiuscolo) | Indiretto (minuscolo) |
|------|---------------------|----------------------|
| Data di completamento pianificata | &quot;Completa il campo **Data di completamento pianificata**.&quot; | &quot;Ogni attività deve avere una data di completamento pianificata.&quot; |
| Modello layout | &quot;In Configurazione, vai a **Modelli di layout**.&quot; | &quot;Assegna un modello di layout a ogni utente.&quot; |
| Amministratore di Gruppo | &quot;Individuare il campo **Amministratore gruppo**.&quot; | &quot;Contattare l&#39;amministratore del gruppo.&quot; |
| Modulo personalizzato | &quot;Allega il **modulo personalizzato** al progetto.&quot; | &quot;Puoi creare un modulo personalizzato per qualsiasi oggetto.&quot; |

### Termini comuni di Workfront (usare l’iniziale maiuscola quando si fa riferimento all’interfaccia utente)

- Modulo personalizzato
- Durata
- Lavoro Necessario
- Data di completamento pianificata
- Tipo di ora
- Amministratore di gruppo/Amministratore di sistema
- Amministratore Workfront (la persona, &quot;amministratore&quot; sempre minuscolo)
- Notifiche promemoria
- Modello layout
- Filtra, Visualizza, Raggruppamento (nel contesto di Filtri, Viste e Raggruppamenti)
- Pianificazione risorse (piani)

### Pulsanti

- Segui maiuscole nell’interfaccia utente
- Se il pulsante è tutto maiuscolo nell’interfaccia utente, scrivilo invece in lettere maiuscole

## Terminologia

### Elementi dell’interfaccia utente

| Elemento | Termine corretto |
|---------|-------------|
| Elenco di opzioni comprimibile | Menu (aggiungi &quot;a discesa&quot; per chiarezza quando necessario) |
| Comandi non disponibili | &quot;unavailable&quot; (non &quot;dimmed&quot;, &quot;disabled&quot; o &quot;grigio&quot;) |
| Popup con campi modulo | Finestra di dialogo |
| Pulsanti senza nome | Utilizza il nome della descrizione del pulsante |
| Messaggi di errore/notifiche popup | &quot;Messaggio&quot; (&quot;Viene visualizzato un messaggio di notifica...&quot;) |
| Input testo | Campo |

### Mansioni

| Contesto | Termine |
|---------|------|
| Il ruolo in Workfront | Amministratore di sistema |
| Persona che amministra Workfront | Amministratore Workfront |

### Sovrascrivi e Sostituisci

Sono parole diverse con significati diversi — usate quella corretta per il contesto.

### Estensioni file

Utilizza sempre l’estensione del file, non l’acronimo: &quot;Carica un file .pdf&quot; (non &quot;Carica un PDF&quot;).

## Formattazione procedura

### Introduzioni

Includere solo quando necessario. Formati preferiti (in ordine):

1. Frase infinita + due punti: &quot;Per creare una password temporanea per un utente nel sistema:&quot;
2. Frase completa + punto: &quot;Devi creare password temporanee per gli utenti nel tuo sistema&quot;.
3. Frase infinita + &quot;segui questi passaggi&quot; + due punti: &quot;Per creare una password temporanea per un utente, segui questi passaggi:&quot;

### Convenzioni dei passaggi

- Utilizzare elenchi numerati per tutte le procedure, anche quelle a passo singolo
- Azioni grassetto selezionabili ed elementi dell’interfaccia utente nei passaggi
- Utilizza verbi di azione specifici: &quot;type&quot;, &quot;select&quot;, &quot;click&quot; — evitare &quot;set&quot; o &quot;specified&quot;
- Descrizione delle risposte di sistema prima o immediatamente dopo

Esempio di posizionamento della risposta del sistema:

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### Passaggi condizionali e facoltativi

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### Passaggi secondari ed elenchi di campi

Introdurre con un comando imperativo (Configura, Specifica, Do), quindi elencare i campi:

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### Metodi multipli

Documenta prima il modo più semplice:

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### Parentesi angolari rette

- Mostra navigazione menu/schede: &quot;Fai clic su **E-mail** > **Notifiche**&quot;
- Nomi dei pulsanti grassetto, non parentesi
- Spazi intorno alle parentesi
- Non utilizzare per il menu principale: utilizza lo snippet del menu principale

## Riferimenti grassetto e corsivo

### Grassetto — Da utilizzare per:

| Contesto | Nei passaggi | Passaggi esterni |
|---------|----------|---------------|
| Pulsanti/collegamenti cliccabili | Grassetto | Non grassetto |
| Nomi menu | Grassetto | Non grassetto |
| Nomi di icone | Grassetto | Non grassetto |
| Nomi di finestre di dialogo | Grassetto | Non grassetto |
| Nomi di pagina | Grassetto | Non grassetto |
| Nomi di campi negli elenchi di campi | Grassetto + due punti | Non grassetto |

### Corsivo: da utilizzare per:

- Testo che l&#39;utente deve digitare: &quot;Digitare *my.workfront.com* nella casella URL&quot;
- Mai per enfasi — ristrutturate la frase

## Note, Suggerimenti, Avvertenze — Linee guida dettagliate

### Quando utilizzarli

**Nota** (`>[!NOTE]`):
- Informazioni che non si adattano al contesto del paragrafo
- Evita frustrazione da parte dell&#39;utente o tempo sprecato (&quot;Per completare questa attività devi disporre di una licenza del team.&quot;)
- Informazioni sul controllo delle versioni o sulla compatibilità (&quot;Questa funzionalità è disponibile solo nel Visualizzatore bozze legacy&quot;).

**Consiglio** (`>[!TIP]`):
- Suggerimenti che semplificano la vita dell’utente ma non sono fondamentali
- Esempio: &quot;È possibile condividere più documenti contemporaneamente tenendo premuto il tasto Maiusc&quot;.

**Importante** (`>[!IMPORTANT]`):
- Informazioni essenziali per la fase o la procedura

**Avviso** (`>[!WARNING]`):
- Avvisa l&#39;utente in caso di potenziale perdita di dati

### Regole

- Un massimo per sezione
- Non impilare mai più note/suggerimenti/avvertenze
- Combinare le note correlate in una con un elenco puntato
- Non utilizzare le note per annunciare nuove funzioni; rielaborare il testo dell&#39;articolo

## Serie di riferimenti incrociati

| Situazione | Pattern |
|-----------|---------|
| Dopo aver fornito le informazioni | &quot;Per ulteriori informazioni, vedere [Nome articolo](link).&quot; |
| Senza informazioni preliminari | &quot;Per informazioni, vedere [Nome articolo](link).&quot; |
| In una sezione di un altro articolo | &quot;Per ulteriori informazioni, vedere [Sezione] in [Nome articolo](link).&quot; |
| A una sezione nello stesso articolo | &quot;Per ulteriori informazioni, vedere [Sezione] in questo articolo.&quot; |

Posizione:
- Nei paragrafi: in linea
- Passaggi della procedura In: nuova riga dopo il passaggio
- Più riferimenti: elenco puntato

## Caratteristiche di qualità DQTI

### Facilità d&#39;uso

| Caratteristica | Domande chiave |
|---------------|---------------|
| Orientamento attività | L’obiettivo è quello di offrire all’utente le funzionalità necessarie? Il titolo è un verbo imperativo? |
| Precisione | La terminologia corrisponde esattamente all’interfaccia utente? I dettagli tecnici sono corretti? |
| Completezza | Ci sono abbastanza informazioni per agire? Sono presenti informazioni non necessarie da rimuovere? |

### Facile da comprendere

| Caratteristica | Domande chiave |
|---------------|---------------|
| Chiarezza | Ogni frase è un&#39;idea? Le parole sono semplici e dirette? |
| Concretezza | Esistono esempi specifici? Gli elementi dell’interfaccia utente sono denominati esattamente? |
| Stile | Voce attiva? Il presente? Terminologia coerente? |

### Facile da trovare

| Caratteristica | Domande chiave |
|---------------|---------------|
| Organizzazione | Struttura logica? Divulgazione progressiva? Se del caso, la più recente? |
| Recuperabilità | Un lettore può analizzare le intestazioni e comprendere il contenuto? Vengono forniti collegamenti? |
| Efficacia visiva | Elenchi per più di 3 elementi? Tabelle per confronti? Nessun paragrafo di muro di testo? |

## Correzioni comuni

| Problema | Correzione |
|-------|-----|
| &quot;Siamo entusiasti di presentare...&quot; | Rimuovi lingua di marketing: specifica direttamente la funzione |
| &quot;La configurazione di X è stata aggiornata&quot; | &quot;È ora possibile configurare X&quot; (attivo, orientato alle attività) |
| &quot;Si consiglia di...&quot; | &quot;Consigliamo...&quot; |
| &quot;Gli utenti possono utilizzare la funzione per...&quot; | &quot;È possibile utilizzare la funzione per...&quot; |
| &quot;imposta il valore&quot; / &quot;specifica l’opzione&quot; | &quot;digita il valore&quot; / &quot;seleziona l’opzione&quot; |
| &quot;disabilitato&quot; / &quot;disabilitato&quot; | &quot;unavailable&quot; |
| &quot;menu a discesa&quot; (autonomo) | menu o menu a discesa |
| lui/lei&quot; | &quot;loro&quot; o ristrutturazione |
